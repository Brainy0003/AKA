Getting Started with C#

class MyFirstClass
{
    static void Main ( )
    { 
       System.Console.Write ( "Hello World!" ) ; 
    }

Beginners are recommended to start by making simple Console Applications for the basic beginning examples. A Console Application is not something you run on a game console like the Xbox or something. Basically a Console Application is something which you would run on old school computers which didn't have a Windows type operating system. A Console application is pretty simple, it maybe has some code which does something like Print some words to the screen. If you want an application which you can run on Windows, you have to include a bunch of extra code in there which communicates with the Windows operating system. So to keep our first code samples as minimal as possible, we will start with simple applications which you run from the Command Line.

If you have installed Visual C# Express, it comes with a nice IDE ( Integrated Development Environment ) which is supposed to make life easier for authoring code and organizing your projects. In this tutorial, we actually have you ignore your nice shiny IDE, and instead we use a dinky little application that sits in a side folder that came along Visual C# Express called csc. Once you get used to it, this is actually a much faster way of doing quick little applications than using the IDE. I recommend doing the first 20 or 30 tutorial examples using the Command Line Compiler, then have a look at Compiling a Console Application using the Visual C# IDE, until you think you have a feel for the C# language and are ready to begin wrapping your head around writing applications that will run on Windows.

Locating The Command Line Compiler

OK, assuming you have already installed Visual C# Express or the .Net Framework SDK from Microsoft, you should now have the ability to author Console Applications, which you can run from the Command Line. To locate the compiler, go to Start / Search and locate the file called csc.exe

Open the folder containing the application csc.exe
You'll want to select and copy the path to the application. Mine looks like this:
C:\WINNT\Microsoft.NET\Framework\v2.0.50727

Don't see the Path?

There are different ways to do that depending on how you have your operating system set up. If you click one time on the application, you should see the path appear in the address bar of the folder and you can copy it right out of there. Some versions of Windows, like Windows XP may have the address bar turned off by default. You have to turn the address bar on in the folder preferences for your operating system. As a programmer, you will need quick access to the path of your files.

If you try to run the application you will get an error message. This is the type of application which accepts Command Line Arguments, meaning you don't just double click on the application and run it, like you would with normal Windows type applications. Instead what you do is you type some commands and hit Enter, and the application will execute whatever you tell it to do.

First we have to Set the Path so Windows know where to find it.

Setting the Path

This section is informative and technical. FYI Beginners can skip over this chapter if you like.

One thing you could do is put your code file in the same folder as csc.exe. Then to compile it, all you would have to do is: from the Command Prompt, go to that folder and type:
csc NameOfFile.cs
and it would compile your code into an application called
NameOfFile.exe
Pretty simple. However, you probably don't want to store your files in the same folder as your application. There's already a ton of files in there that the compiler needs, so it will get confusing pretty fast.

As long as you start in the same directory as csc.exe, you can invoke it by just typing csc
If you start from another directory like My Documents, you have to invoke the compiler by specifying the path to it before the name.
For instance, if I were starting from the main C: Drive, to invoke the compiler, I would enter:
\WINNT\Microsoft.NET\Framework\v2.0.50727\csc FileName.cs
This will compile FileName.cs into FileName.exe as long as it's in the same folder as csc.exe.
Likewise you can specify the path to the directory where your file is stored. Say my C# code file was in My Documents:
\Documents and Settings\UserName\My Documents\FileName.cs
Now we have a long-ass command that looks like this:
\WINNT\Microsoft.NET\Framework\v2.0.50727\csc \Documents and Settings\UserName\My Documents\FileName.cs
Rather than having to go around adding these long pathnames every time we want to call the compiler, we can actually tell Windows where to find the application csc.exe one time, and from then on we can simply call csc by name regardless of what directory we are starting in.

Create a new Directory from the Command Prompt

Make a Directory to store your code in:
Type
md C#
and hit Enter. This creates a new folder called C#. Type dir to verify that the new directory is there.

Open specified directory

To open the C# directory:
Type
cd c#
You will see the command prompt change to show that you are now in the C# directory.

Inside that directory, let's make another directory for our first project called Hello:
Type
md Hello
then type dir to verify that the new Hello directory was created.

Open the Hello directory:
Type
cd hello

Create your first code

Type
notepad hello.cs
Notepad opens and ask if it's OK to create a new file. Just hit Enter.
Type the following code:

Go to File / Save As
Save the file hello.cs in the Hello folder you created earlier located at My Computer / Hard Drive ( C:/C#/Hello ). From the Command Prompt, you should now be able to type dir and see your newly created file: hello.cs

Compile the code

To compile your code: from the Command Prompt type:
csc hello.cs
If all went well your program should compile without any errors. You should find the newly created program called hello.exe in the Hello folder. Type dir and you can see your new program.

Execute the Application

To run the application hello.exe from the Command Line, type:
hello
The program executes and prints the message to the Command Prompt window. Nothing spectacular, but congratulations! you're a programmer. Big whoop. Now get back to work.

Note that once your have entered a few commands into the Command Prompt window, you can hit the Up or Down Arrow keys to recall any commands you already entered. So once you have compiled one program, if you need to compile it a second time, you just arrow up till you find the command from the previous compile and hit Enter. Now hit the Arrow Down key and there's the name of the file. Hit Enter to run the program again. Saves typing.

Compiler Command Line Reference

Compiles File.cs producing File.exe:
csc File.cs
Compiles File.cs producing File.dll:
csc /target:library File.cs
Compiles File.cs and creates My.exe:
csc /out:My.exe File.cs
Compile Application with a space in the name:
csc /out:"My Application".exe hello.cs
Compile application without flashing the Microsoft Logo
csc /nologo hello.cs
Compile Application with a different name, No Logo
csc /nologo /out:My_Application.exe hello.cs
Compiles all of the C# files in the current directory, with optimizations on and defines the DEBUG symbol. The output is File2.exe:
csc /define:DEBUG /optimize /out:File2.exe *.cs
Compiles all of the C# files in the current directory producing a debug version of File2.dll. No logo and no warnings are displayed:
csc /target:library /out:File2.dll /warn:0 /nologo /debug *.cs
Compiles all of the C# files in the current directory to Something.xyz (a DLL):
csc /target:library /out:Something.xyz *.cs
To compile and generate XML documentation within
csc.exe /out:CSharpProgram.exe CSharpProgram.cs /doc:CSharpProgram.xml
