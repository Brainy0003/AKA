I tried the free Visual C++ download from Microsoft and found that none of the tutorials I could find on the web for Visual C++ were compatible, mostly because of changes to the interface. This is Visual C++ Express and not only were there no tutorials yet on the web, but a lot of the built in help feature had broken links, all the help seemed to assume you are an experienced programmer and was extremely technical and there was no "help for beginner's" feature, so although you can still compile your programs with Visual C++ O.K. from the command line, I ended up using Borland's free command line compiler, so I recommend you download that one as well to follow along.

http://www.borland.com/products/downloads/download_cbuilder.html

Once you have installed your free Borland compiler we will go straight into the forbidden zone known as DOS. First you want to open your Command Prompt.
Assuming you are on a PC and have no idea what a Command Prompt or DOS is:
On Windows, go to Start / All Programs / Accesories / Command Prompt. Remember to click on the Compilers button in your menu bar to return to this tutorial.

If you blow the window up to full screen size you'll get an idea of what computers looked like before there was Windows, everything was done at the text level, and that's where we're going to be spending a lot of time from now on.

Depending on your computer you may see something a little different, mine looks like this:

We want to start at the root directory like this:

C: represents your main hard drive. Yours may be a different letter, D: perhaps.
If yours already looks like this, that's fine. If yours looks more like mine with a couple of folder names after it
Type the following:
cd..
and hit Enter. (That's CD period period)
This should move you up one Directory:

Type CD.. again and hit Enter. This should move you up another level.
Continue entering CD.. until you can go no higher. You have reached the top level of your Hard Drive.

Type
dir
and hit Enter.
This should display the contents of your hard drive:

You can see there may be one or two files and a number of folders, or Directories.
If you have installed the Borland command line compiler, you should see a Directory named "Borland".
Type
cd borland
This will take you to the Borland directory, or folder.
Enter dir to display the contents of that directory. There is only one directory contained therein: BCC55.
Enter cd bcc55, then dir to show the contents of the BCC55 directory.

Create a new Directory from the Command Prompt

Make a directory to store your code in:
Type
mkdir Projects
and hit Enter. This creates a new folder called Projects. Type dir to verify that the new directory is there.

Open specified directory

The Borland compiler is in the directory called Bin (short for Binaries).
To open the Bin directory:
Type
cd../bin
This will take you up one directory, and then down again all in one command. The command prompt should indicate that you are in the Bin directory. Type dir to view the contents.

Run a program from the Command Prompt

The Borland C++ compiler application is called bcc32.exe. You execute the program by typing a command from the prompt, such as telling it to compile a piece of code for you. You do not actually run the application. If you do try to run the application, it will dump a list of commands to the screen. These are the various commands you use to tell the compiler what to do.
To run the Borland compiler:
Type
bcc32
and hit Enter.

Create your first code

Type
notepad hello.cpp
Notepad opens and ask if it's O.K. to create a new file. Just hit Enter.
Type the following code. Yes, actually type it. Don't just copy it out of the box . Put your lazy fingers on the keys and type.

#include <stdio.h>
int main ( void )
{
         printf ( "Hello" ) ;
return 0 ;
}

Go to File / Save As
Save the file hello.cpp in the Projects folder you created earlier located at My Computer / Hard Drive ( C: / Borland / BCC55 / Projects ).

Compile the code

Now copy the next command if you like: (highlight the text in the box and go to Edit / Copy in your brower menu)

bcc32 -If:\Borland\BCC55\Include -Lf:\Borland\BCC55\Lib -n\Borland\BCC55\Projects \Borland\BCC55\Projects\hello.cpp

Of course that's assuming you are using the suggested Borland compiler and you installed it in the default folder as I did, which is the main hard drive. If not you will need to make the necessary adjustments to the path: ( -If:\YourCompiler'sDirectory\Include etc.)

Now right click in the Command Prompt window and select Paste. Then hit Enter.
If all went well your program should compile without any errors. You should find the newly created program called hello.exe in the Projects folder. To get there, navigate up out of the Bin directory and down into the Projects directory by typing cd../projects. Type dir and you should see your new program. To run it, type
hello
The program executes and prints the word Hello to the Command Prompt window. Nothing spectacular, but congratulations! you're a programmer. Big whoop. Now get back to work.

If your program didn't compile successfully, proceed directly to Debugging.

First let's discuss how we called the compiler and then we'll talk about our code. While at first glance it looks like a long, intimidating string of complicated code, but the command to the compiler is fairly simple when you break it down.
First we call the compiler:
bcc32
Remember earlier if we just call the compiler it prints out a list of commands? If we call the compiler followed by various parameters, it will execute them. Here we issue 3 parameters which tell the compiler where to find some necessary files. Some are in the Include directory located at C:\Borland\BCC55\Include\, some are in the Lib directory. ( -If for Include files -Lf for Library files ) The path is given so the compiler knows where to find these files. The command -n tells the compiler where to put the finished program: in the Projects directory and lastly, the path to the source code itself is given: \Borland\BCC55\Projects\hello.cpp. If you take away the long path info, it's not so intimidating. You can save the command in a text file for later use. Enter notepad compiler_commands.txt into the Command Prompt. Paste the following shortcut:

bcc32 -If:\Borland\BCC55\Include -Lf:\Borland\BCC55\Lib -n\Borland\BCC55\Projects \Borland\BCC55\Projects\.cpp

Now when you want to compile another program you can open the page, insert the name of the .cpp file, copy and you're ready to paste into the command prompt and compile another program.

Note that once your have entered a few commands into the Command Prompt window, you can hit the up or down arrow keys to recall any commands you already entered. So once you have compiled one program and you want to compile another, you just arrow up till you find the command from the previous compile. Using the left arrow key, scroll the blinking cursor until it is underneath the period. Now hit the Backspace key to delete the name of the previous file and enter the new one.
